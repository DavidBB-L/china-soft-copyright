# 📄 中国软著申请材料生成

一个 [Hermes Agent](https://github.com/nousresearch/hermes-agent) 技能（Skill），用于生成符合中国软件著作权登记要求的申请材料。

## ✨ 功能一览

| 功能 | 说明 |
|------|------|
| 📝 **申请引导** | 22 个字段逐步确认，避免填错返工 |
| 💻 **源代码文档** | 自动生成 50 行/页的 `.txt` 文件（前30+后30 或全部） |
| 📄 **软件说明书** | Markdown → PDF，≥30 行/页 |
| 📋 **申请表指导** | 每个字段的正确填写方式 + 字数限制提示 |

## 🚀 快速开始

### 安装

```bash
git clone https://github.com/249695811/china-soft-copyright.git ~/.hermes/skills/china-soft-copyright
```

### 使用方式

安装后，直接对你的 Hermes Agent 说：
- "我要申请软著"
- "帮我准备软著材料"
- "生成源代码文档"

Agent 会自动引导你完成全部流程。

## 📋 工作流程

### Step 1：基本信息采集

依次确认以下内容：

| 字段 | 说明 | 注意事项 |
|------|------|---------|
| 软件全称 | 须以"软件/APP/系统/平台"结尾 | ≤15 字，用最宽泛的领域词 |
| 英文名 | 可选，建议添加 | 格式：`中文名 英文名 软件类型` |
| 版本号 | 首次申请填 V1.0 | 格式：V1.0 或 1.0 |
| 著作权人 | 身份证姓名或营业执照全称 | — |
| 开发方式 | 独立/合作/委托/下达任务 | 最常见：独立开发 |
| 权利范围 | 全部权利 / 部分权利 | 最常见：全部权利 |
| 首次发表 | 未发表 / 已发表 | 首次申请选"未发表" |

### Step 2：技术信息采集

| 字段 | 字数限制 | 示例 |
|------|---------|------|
| 源程序量 | 无限制 | 精确行数，Agent 帮你 `wc -l` 统计 |
| 软件分类 | — | 默认 `30200-0000`（应用软件） |
| 开发完成日期 | — | 须早于申请日，与 Git 日志一致 |
| 开发硬件环境 | ≤50 字 | `Intel Core i7 CPU, 16GB内存, 512GB固态硬盘` |
| 运行硬件环境 | ≤50 字 | `Intel i3 及以上CPU, 2GB内存, 100MB磁盘` |
| 开发操作系统 | ≤50 字 | `Windows 10及以上, macOS 12及以上` |
| 开发工具/环境 | ≤50 字 | `VS Code, Node.js v18, Electron 33, Git` |
| 运行平台 | ≤50 字 | `Windows 10/macOS 12/Linux, Chrome/Firefox/Safari` |
| 支撑环境 | ≤50 字 | `Node.js v18及以上, Chromium浏览器内核` |
| 编程语言 | 无限制 | 列出所有语言：`JavaScript, HTML, CSS, JSON` |
| 面向领域 | ≤50 字 | 如：教育、医疗、金融、电商、内容创作 |
| 开发目的 | ≤50 字 | 一句话说明为什么做这个软件 |
| **主要功能** | **500-1300 字** | ⚠️ **审核重点**，须与说明书一致 |
| **技术特点** | **≤100 字** | ⚠️ 必须精炼，体现独创性 |

### Step 3：材料生成

信息确认完毕后，按顺序生成：

1. **源代码文档** (`.txt`) — 50 行/页，行号 + 代码
2. **软件说明书** (`.pdf`) — ≥30 行/页，含功能介绍 + 截图
3. **申请表填写指导** — 逐字段列出正确填写方式

## 📏 提交规则

| 材料 | 页数规则 | 每页行数 | 格式要求 |
|------|---------|---------|---------|
| 程序鉴别材料 | ≥60 页→前30+后30页；<60 页→全部 | ≥50 行/页 | 行号+代码，等宽字体 |
| 文档鉴别材料 | ≥60 页→前30+后30页；<60 页→全部 | ≥30 行/页 | 说明书/用户手册/设计文档 |

## ⚠️ 常见坑（必读）

1. **软件全称不能改** — 提交后改名等于重新申请
2. **命名选宽泛词** — "内容创作"比"短视频"好，"写作工具"比"剧本编辑器"好
3. **主要功能 500-1300 字是审核重点** — 必须与说明书内容一致
4. **技术特点只有 100 字** — 提前精炼好，别写多了被截断
5. **各 50 字字段别超限** — 表单会直接截断，超了不提示
6. **版本号不影响保护范围** — V1.1 的软著覆盖后续所有版本
7. **重构不影响软著** — 改内部代码不影响证书有效性
8. **2026年3月15日起使用新版申请表** — 旧版已废止

## 📐 PDF 生成参数

经过多次调试的参数，不要随意改动：

| 类型 | 字体大小 | 行高 | 边距 | 效果 |
|------|---------|------|------|------|
| 源代码 PDF | 8.5px | 1.18 | 0.5cm | 50 行/页 |
| 说明书 PDF | 9.8px | 1.55 | 1.3cm | ≥30 行/页 |

使用 **Node.js Playwright API** 生成 PDF，不要用 Chromium CLI 的 `--print-to-pdf`（会忽略 `@page` CSS 导致行数不可控）。
