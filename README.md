# 📄 中国软著申请材料生成

一个 [Hermes Agent](https://github.com/DavidBB-L/china-soft-copyright) 技能（Skill），用于生成符合中国软件著作权登记要求的申请材料。

## ✨ 功能一览

| 功能 | 说明 |
|------|------|
| 🔍 **自动扫描** | AI 扫描项目代码，自动推断软件信息（名称、功能、技术栈） |
| 💻 **源代码文档** | 自动生成 50 行/页的 `.txt` 文件（前30+后30 或全部） |
| 📄 **软件说明书** | Markdown → PDF，≥30 行/页 |
| 📋 **申请表指导** | 每个字段的正确填写方式 + 字数限制提示 |
| 🔎 **AIGC痕迹检测** | 检测材料中的AI生成痕迹，避免驳回风险 |
| ✅ **提交前检查** | 10项逐条核对清单，防止低级错误 |

## 🚀 快速开始

### 安装

```bash
git clone https://github.com/DavidBB-L/china-soft-copyright.git ~/.hermes/skills/china-soft-copyright
```

### 使用方式

安装后，直接对你的 Hermes Agent 说：
- "我要申请软著"
- "帮我准备软著材料"
- "生成源代码文档"

Agent 会自动扫描项目、读代码、生成材料，用户只需确认少量信息。

## 📋 工作流程

### Step 1: 扫描项目（自动）

AI 扫描项目目录，列出源文件、统计行数、读取 README 和配置文件。

### Step 2: 理解项目（自动）

AI 读取关键源文件，自动推断软件全称、版本号、编程语言、技术栈、主要功能、技术特点等。

### Step 3: 只问用户必要信息

AI 只问**确实无法从代码推断**的信息：
1. **著作权人** — 必须问（身份证姓名或公司全称）
2. **软件全称** — 如果能推断就建议一个，让用户确认
3. **版本号** — 如果能读到就自动填，否则默认 V1.0
4. **开发完成日期** — 从 git log 最早 commit 建议

### Step 4: 生成材料（自动）

1. **源代码文档** (`.txt`) — 50 行/页，行号 + 代码
2. **软件说明书** (`.pdf`) — ≥30 行/页，含功能介绍
3. **申请表填写指导** — 逐字段列出正确填写方式

### Step 5: AIGC 检测

对生成的材料进行 AIGC 痕迹检测，发现问题直接修正。

## 📏 提交规则

| 材料 | 页数规则 | 每页行数 | 格式要求 |
|------|---------|---------|---------|
| 程序鉴别材料 | ≥60 页→前30+后30页；<60 页→全部 | ≥50 行/页 | 行号+代码，等宽字体 |
| 文档鉴别材料 | ≥60 页→前30+后30页；<60 页→全部 | ≥30 行/页 | 说明书/用户手册/设计文档 |

## ⚠️ AI 承诺条款（2026年3月15日起施行）

新版申请表签章页新增手抄承诺：申请人须声明**"未使用AI开发编写代码、撰写文档或生成登记申请材料"**，并手抄失信+征信惩罚条款。

这是**二元判断**——不看比例、不看贡献度，只问"用了还是没用"。

详见 `references/ai-commitment-clause.md`。

## 🔎 AIGC 痕迹检测

材料中的AI痕迹可能导致驳回甚至征信问题。常见痕迹包括：

- **品牌名泄露**：AI模型名（GPT/Claude等）、工具名（LM Studio/Ollama）
- **字数标注**：`（57字）`、`（共800字）`
- **AI自检汇报**：`"不含GPT/Claude"`（把名字写出来了）
- **句式同质化**：每个功能模块的描述骨架完全一致
- **AI填充虚数**：`"六种格式"`但实际只有四种
- **编造测试条目**：`"弱网环境下能正常降级"`但没有具体场景

详见 `references/aigc-detection.md`。

## ⚠️ 常见坑（必读）

1. **软件全称不能改** — 提交后改名等于重新申请
2. **命名选宽泛词** — "内容创作"比"短视频"好，"写作工具"比"剧本编辑器"好
3. **主要功能 500-1300 字是审核重点** — 必须与说明书内容一致
4. **技术特点只有 100 字** — 提前精炼好，别写多了被截断
5. **各 50 字字段别超限** — 表单会直接截断，超了不提示
6. **版本号不影响保护范围** — V1.1 的软著覆盖后续所有版本
7. **重构不影响软著** — 改内部代码不影响证书有效性
8. **2026年3月15日起使用新版申请表** — 旧版已废止，新增AI承诺条款
9. **⚠️ 提交前检查字数标注** — 复制到申请表的内容可能残留 `（57字）` 等标注
10. **⚠️ 说明书功能必须和代码核实** — 不确定的功能宁可不写，也不能编

## 📐 PDF 生成参数

经过多次调试的参数，不要随意改动：

| 类型 | 字体大小 | 行高 | 边距 | 效果 |
|------|---------|------|------|------|
| 源代码 PDF | 8.5px | 1.18 | 0.5cm | 50 行/页 |
| 说明书 PDF | 9.8px | 1.55 | 1.3cm | ≥30 行/页 |

使用 **Node.js Playwright API** 生成 PDF，不要用 Chromium CLI 的 `--print-to-pdf`（会忽略 `@page` CSS 导致行数不可控）。
