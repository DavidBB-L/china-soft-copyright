# 软著申请表字段字数限制（2026年6月调研）

## 字数限制速查表

| 字段 | 字数限制 | 备注 |
|------|---------|------|
| 软件全称 | 一般15字以内 | 须以"软件/APP/系统/平台"结尾 |
| 软件简称 | 不超过全称长度 | 可不填 |
| 版本号 | 无限制 | 格式：V1.0 或 1.0 |
| 源程序量 | 无限制 | 填精确行数 |
| 开发目的 | 无明确限制 | 1-2句话即可 |
| 面向领域/行业 | 无明确限制 | 简短描述 |
| 主要功能 | **500-1300字** | 需与说明书一致，审核重点 |
| 技术特点 | **≤100字** | 必须精炼，体现独创性 |
| 硬件环境 | 无明确限制 | 简短描述 |
| 软件环境 | 无明确限制 | 简短描述 |
| 编程语言 | 无明确限制 | 列出所有语言 |

## 2026年3月15日更新

中国版权保护中心更新了申请表：
- 功能说明部分有扩展
- 签章环节新增诚信说明要求
- 旧版申请表已废止

## 关键发现

1. **主要功能（500-1300字）是审核重点**：必须与说明书内容一致
2. **技术特点只有100字**：提前精炼好，别写多了被截断
3. **各50字字段别超限**：表单会直接截断，超了不提示
4. **软件全称建议15字以内**：以"软件/APP/系统/平台"结尾为佳，但非硬性规定（以"工具"结尾也有通过受理的案例）
5. **⚠️ 提交后无法修改**：在线填写后一旦提交，字段内容无法修改，只能等审核员打回来再改（2026-06实测确认）
6. **⚠️ 复制粘贴要检查残留**：从AI工具复制到申请表的内容可能带字数标注（"（57字）"）、分类标签前缀等，提交前逐字段肉眼检查
